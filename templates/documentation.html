Loan Prediction API Documentation

1. API Overview

The Loan Prediction API allows users to predict whether a loan application will be approved based on the applicant's data. It accepts user input through a POST request to the /predict endpoint and returns the loan approval prediction.

2. API Endpoints

GET /

Returns a welcome message and a link to the documentation page.

{
    "message": "Welcome to the Loan Prediction API! Use the /predict endpoint to make predictions."
}

POST /predict

Accepts input data and predicts whether the loan will be approved.

Input Parameters

Field Type Description
Gender string Applicant's gender. Options: Male, Female.
Married string Applicant's marital status. Options: Yes, No.
Dependents string Number of dependents. Options: 0, 1, 2, 3+.
Education string Applicant's education level. Options: Graduate, Not Graduate.
Self_Employed string Employment status. Options: Yes, No.
ApplicantIncome float Applicant's income (in thousands). Example: 5 (for 5000).
CoapplicantIncome float Co-applicant's income (in thousands). Example: 1.5 (for 1500).
LoanAmount float Loan amount requested (in thousands). Example: 200 (for 200000).
Loan_Amount_Term int Loan repayment term (in months). Example: 360.
Credit_History int Credit history. Options: 1 (good credit), 0 (bad credit).
Property_Area string Property location. Options: Urban, Semiurban, Rural.

3. Example Request

Request JSON

{
    "Gender": "Male",
    "Married": "Yes",
    "Dependents": "0",
    "Education": "Graduate",
    "Self_Employed": "No",
    "ApplicantIncome": 5.0,
    "CoapplicantIncome": 2.5,
    "LoanAmount": 200.0,
    "Loan_Amount_Term": 360,
    "Credit_History": 1,
    "Property_Area": "Urban"
}

Example cURL Request

curl -X POST http://127.0.0.1:5000/predict \
-H "Content-Type: application/json" \
-d '{
    "Gender": "Male",
    "Married": "Yes",
    "Dependents": "0",
    "Education": "Graduate",
    "Self_Employed": "No",
    "ApplicantIncome": 5.0,
    "CoapplicantIncome": 2.5,
    "LoanAmount": 200.0,
    "Loan_Amount_Term": 360,
    "Credit_History": 1,
    "Property_Area": "Urban"
}'

Example Response

{
    "prediction": "Approved"
}

4. Error Handling

The API will return appropriate error responses if there are missing or invalid fields.

Example Error Response

{
    "error": "Invalid input format: Missing 'Gender' field"
}

5. Input Options

The following are the valid options for categorical fields:

  • Gender: Male, Female
  • Married: Yes, No
  • Dependents: 0, 1, 2, 3+
  • Education: Graduate, Not Graduate
  • Self_Employed: Yes, No
  • Credit_History: 1 (good credit), 0 (bad credit)
  • Property_Area: Urban, Semiurban, Rural

6. Data Units

ApplicantIncome, CoapplicantIncome, and LoanAmount must be in thousands.

  • For example, 5000 should be entered as 5.0
  • For example, 200,000 should be entered as 200.0

7. Testing the API

You can use tools like Postman, cURL, or your web app form to test the API. Ensure all required fields are provided to avoid errors.
